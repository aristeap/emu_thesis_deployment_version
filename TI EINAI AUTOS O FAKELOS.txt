Στο example-project (αυτό είναι το copy του) δοκίμασα μία άλλη προσέγγιση. Πρώτα ξεκίνησα να φτιάχνω ένα angular project (γιατί το emu-sdms δεν είναι), έφτιαξα την Angular να είναι  έκδοση 8 
(και να ταιριάζουν και τα υπόλοιπα dependencies μεταξύ τους) και μετά έκανα copy-paste όλα τα αρχεία της emu. Μετά είπα θα πάω σε έναν έναν error και 
θα φτιάχνω ότι μου γράφει οτί είναι λάθος. Το πλάνο είναι να μου βγάζει τα outdated πράγματα ώστε να τα άλλαζα ένα ένα σε κάθε file. 
Αρχικά είχα δοκιμάσει με το emuThesis-vs-update-effort-NEW να κάνω ένα ένα τα αρχεία σε νεότερη έκδοση, αλλά το chatgpt μου άλλαζε και τα ονόματα και πολλά πράγματα στα αρχεία.
Το emuThesis_BACKUP_leitourgei-NEW είναι το αρχείο από το οποίο ένα copy-paste τα αρχεία.Το file αυτό είναι "λειτουργικό", που σημαίνει ότι όταν το κάνω build και μετά start αυτό θα 
ανοίξει την εφαρμογή της emu--και επιπλέον την σελίδα Add Metadata που έχω φτιάξει--. Επειδή είναι λειτουργικό,φτιάχνω και άλλα copy από αυτό.

Στο emuThesis_BACKUP_leitourgei-NEW-Copy θα προσπαθήσω να δω την σειρά με την οποία εκτελούνται τα αρχεία του emu-webapp ώστε να μπορέσω να καταλάβω πως θα κάνω το index του example-project.
Ο τρόπος με τον οποίο θα το κάνω αυτό είναι να προσθέσω ένα node.js script (απί chatgpt) το οποίο θα βάζει σε κάθε αρχείο console.logs που θα λέει τι αρχείο εκτελείται.
